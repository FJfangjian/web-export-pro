# 2026-07-10

## 新增 Word/PDF 导出支持

### 改动文件
- `manifest.json`: 名称改为"网页内容导出工具"，版本升至 1.1.0，描述加入 Word/PDF
- `content/content.js`:
  - 新增 `exportTableAsWord(name, tableData)` — 生成 HTML 格式 .doc（application/msword MIME），Word 可直接打开
  - 新增 `buildHtmlDoc(name, tableData)` — 生成带样式的 Word 兼容 HTML 文档
  - 新增 `printAsPDF(name, htmlContent)` — 通过浏览器打印窗口触发 PDF 导出（window.open + window.print）
  - 新增 `exportTableAsPDF(name, tableData)` — PDF 导出封装
  - 新增 `downloadBlob(filename, blob)` — Blob 下载通用函数
  - `showExportPanel` 面板底部新增格式选择器（xlsx/csv/doc/pdf）
  - `exportSingleTable` 支持 format 参数，分发到对应导出函数
  - `showVisibleContentPreview` 格式下拉新增 .doc Word / PDF 文档
  - `exportVisibleContent` 处理 doc/pdf 格式（优先使用视觉表格，否则用表头+数据）
- `content/content.css`: 新增 `.hte-table-format-bar` 和 `.hte-format-select` 样式
- `popup/popup.html`: 标题和副标题更新，按钮文案调整
- `popup/popup.js`: 无需改动（通信逻辑不变）

### 技术决策
- **Word 导出**：直接生成带 CSS 样式的 HTML 文档，以 `application/msword` MIME 保存为 .doc。Word 能完美渲染表格样式，无需额外依赖库。
- **PDF 导出**：利用浏览器原生打印功能（`window.open` + `window.print`），用户在打印对话框中选择"另存为 PDF"。避免了 jsPDF 等重型库的引入。
- 表格导出面板和可见内容面板均支持全部四种格式。
